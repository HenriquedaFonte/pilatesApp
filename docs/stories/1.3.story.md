# Story 1.3 — Frontend: "Desativar" como Ação Primária + Fluxo de Soft Delete

**Epic:** 1 — Proteção de Dados (Fase 1 — MUST)
**Status:** Done
**Estimativa:** S (2–3h)
**Tipo:** Frontend (React / Supabase)
**Depende de:** Story 1.1 (colunas `deleted_at`/`deleted_by` em profiles), Story 1.2 (edge function com soft delete)

---

## User Story

**Como** professora do estúdio,
**Quero** que o botão "Inativar Aluno" seja a ação principal ao precisar remover um aluno temporariamente, e que "Excluir" resulte em soft delete (não hard delete),
**Para que** eu tenha uma camada de proteção contra exclusões acidentais irreversíveis.

---

## Contexto para o Dev

Arquivo principal: `src/pages/TeacherStudents.jsx`

Situação atual no `DropdownMenu` de cada aluno:
1. Observações
2. Resetar Senha
3. Inativar/Ativar Aluno (já implementado via `is_active`)
4. **[Separador]**
5. Excluir Aluno → chama `manage-student` com `action: 'delete'` (agora soft delete após Story 1.2)

**Mudanças desta story:**
- O botão "Excluir Aluno" agora chama `action: 'delete'` (que após Story 1.2 faz soft delete)
- Texto do botão: "Excluir Aluno" → "Mover para Lixeira"
- O modal de confirmação de exclusão atualiza texto para indicar que é reversível
- A query de listagem de alunos deve filtrar `deleted_at IS NULL`

---

## Acceptance Criteria

- [ ] **AC-1:** A listagem de alunos em `TeacherStudents.jsx` filtra apenas alunos com `deleted_at IS NULL` (alunos em soft delete não aparecem na lista normal)
- [ ] **AC-2:** O item "Inativar Aluno" continua no dropdown, mas é reposicionado como a ação recomendada (acima do separador, com destaque visual leve)
- [ ] **AC-3:** O item "Excluir Aluno" passa a se chamar "Mover para Lixeira" com ícone `Trash2`
- [ ] **AC-4:** O modal de confirmação de "Mover para Lixeira" informa que a ação **pode ser revertida em até 90 dias** pelo painel de alunos excluídos
- [ ] **AC-5:** O handler `handleDeleteStudent` continua invocando `manage-student` com `action: 'delete'` (sem mudança no lado do cliente — a mudança de comportamento está na edge function)
- [ ] **AC-6:** Nenhum botão de "hard delete" permanente é exposto na UI nesta story (será adicionado na Story 2.2)

---

### OUT
