# Story 1.1 — Infraestrutura de Banco de Dados: Audit Log + Soft Delete

**Epic:** 1 — Proteção de Dados (Fase 1 — MUST)
**Status:** InReview
**Estimativa:** M (3–5h)
**Tipo:** Database + Migration

---

## User Story

**Como** professora do estúdio,
**Quero** que toda exclusão de aluno seja registrada permanentemente e que o aluno seja movido para um estado "excluído" em vez de removido do banco,
**Para que** eu possa rastrear quem fez o quê e recuperar dados em caso de erro.

---

## Contexto para o Dev

Esta é a story de **fundação**. Ela cria as bases de banco de dados que as stories 1.2 e 1.3 dependem. Não há mudanças de UI ou edge function aqui — apenas migrations SQL seguras (apenas ADD COLUMN / CREATE TABLE).

## Acceptance Criteria

- [ ] **AC-1:** Tabela `audit_log` existe no banco com as colunas: `id`, `actor_id`, `actor_email`, `action`, `target_id`, `target_name`, `target_email`, `snapshot_data` (JSONB), `created_at`
- [ ] **AC-2:** `audit_log` tem RLS configurada: INSERT permitido via service role apenas, DELETE e UPDATE bloqueados para todos (append-only)
- [ ] **AC-3:** Tabela `profiles` tem as colunas `deleted_at TIMESTAMPTZ` e `deleted_by UUID` (nullable, ADD COLUMN IF NOT EXISTS — sem risco de quebrar dados existentes)
- [ ] **AC-4:** Migration é reversível: comentário com instrução de rollback incluída no arquivo SQL
- [ ] **AC-5:** Nenhuma coluna ou tabela existente é modificada ou renomeada

## Riscos

| Risco | Probabilidade | Impacto | Mitigação |
|-------|---------------|---------|-----------|
| Migration falha em produção por conflito de nome | Baixa | Alto | Usar `IF NOT EXISTS` em todas as instruções — sem risco de erro se rodar duas vezes |
| RLS mal configurada permite DELETE em audit_log | Média | Alto | AC-2 exige validação explícita; testar com role anon/authenticated tentando DELETE |
| ADD COLUMN em `profiles` causa lock em produção | Baixa | Médio | PostgreSQL ADD COLUMN com default NULL é instantâneo (sem rewrite de tabela) |
| Rollback necessário em emergência | Baixa | Alto | Cada migration inclui comentário com instrução DROP — executar manualmente se necessário |

## Dev Notes

### Stack e Localização de Arquivos
- Banco: **Supabase (PostgreSQL)** — Free plan
- Migrations: `supabase/migrations/` — naming convention: `YYYYMMDDHHMMSS_descricao.sql`
- RLS: Padrão existente em `supabase/migrations/20241011044500_add_essential_rls_policies.sql`
- Coluna `is_active` já existe em `profiles` (migration `20260531000000_add_is_active_to_profiles.sql`)
- Colunas `deleted_at` e `deleted_by` são **novas** — sem risco de conflito

### Estrutura da Tabela `audit_log`
```sql
CREATE TABLE IF NOT EXISTS public.audit_log (
  id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  actor_id     UUID NOT NULL,                    -- quem executou (auth.users.id)
  actor_email  TEXT NOT NULL,                    -- snapshot do email do executor
  action       TEXT NOT NULL,                    -- ex: 'student.delete', 'student.soft_delete'
  target_id    UUID,                             -- UUID do aluno afetado
  target_name  TEXT,                             -- snapshot do nome
  target_email TEXT,                             -- snapshot do email
  snapshot_data JSONB,                           -- dados completos do perfil no momento
  created_at   TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

### RLS da `audit_log`
```sql
-- Habilitar RLS
ALTER TABLE public.audit_log ENABLE ROW LEVEL SECURITY;

-- Apenas service role pode inserir
CREATE POLICY "service_role_insert_audit_log"
  ON public.audit_log FOR INSERT
  TO service_role
  WITH CHECK (true);

-- Nenhuma role pode UPDATE ou DELETE (append-only)
-- Leitura: apenas teacher logado
CREATE POLICY "teacher_select_audit_log"
  ON public.audit_log FOR SELECT
  USING (
    EXISTS (
      SELECT 1 FROM public.profiles
      WHERE id = auth.uid() AND role = 'teacher'
    )
  );
```

### ADD COLUMN em `profiles`
```sql
ALTER TABLE public.profiles
  ADD COLUMN IF NOT EXISTS deleted_at TIMESTAMPTZ,
  ADD COLUMN IF NOT EXISTS deleted_by UUID REFERENCES auth.users(id);
```

### Indexes recomendados
```sql
CREATE INDEX IF NOT EXISTS audit_log_target_id_idx ON public.audit_log(target_id);
CREATE INDEX IF NOT EXISTS audit_log_created_at_idx ON public.audit_log(created_at DESC);
CREATE INDEX IF NOT EXISTS profiles_deleted_at_idx ON public.profiles(deleted_at) WHERE deleted_at IS NOT NULL;
```
